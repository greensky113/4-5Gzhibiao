# 4G5G指标自动通报 - 详细使用说明书

## 一、技能简介

### 1.1 功能概述

4G5G指标自动通报是一个全自动处理4G/5G无线网络指标数据的技能，能够：
- 自动读取原始Excel数据文件
- 生成汇总表
- 按规则计算各项指标
- 生成可视化看板（PNG图片）

### 1.2 适用场景

- 节假日景区网络指标监控通报
- 重大活动通信保障指标统计
- 日常网络性能分析报告

### 1.3 调用方式

在MobileClaw中直接呼叫：
```
执行4G5G指标自动通报
```

## 三、源数据格式要求

### 3.1 文件命名规则

- 文件格式：`.xlsx`（Excel 2007及以上）
- 临时文件自动过滤（以 `~$` 开头的文件会被忽略）

### 3.2 Excel Sheet结构

每个源数据Excel文件必须包含以下Sheet：

#### Sheet 1：汇总表

| 列序 | 4G列名 | 5G列名 | 数据类型 |
|------|--------|--------|----------|
| 1 | 时间 | 时间 | datetime |
| 2 | LTE小区 | NR小区 | string |
| 3 | 存在 | 存在 | number |
| 4 | 小区内的最大用户数 | 用户数-SA | number |
| 5 | 总流量GB | 5G流量(GB)-集团 | number |
| 6 | 下行PRB利用率(%) | 下行PRB平均利用率(%)-集团 | number |
| 7 | 上行PRB利用率(%) | 上行PRB平均利用率(%)-集团 | number |
| 8 | VoLTE用户数 | VoNR用户数 | number |
| 9 | RRC连接态最大用户数(个) | 小区内最大RRC连接态用户数 | number |

#### Sheet 2+：各小区组明细

以小区组名作为Sheet名称（如"永胜山地自行车比赛"、"华坪县中心镇龙舟比赛"），包含该小区组下所有小区的详细指标数据。

### 3.3 数据时间范围

- 支持多个时间点（建议覆盖0:00-8:00，每小时一个时间点）
- 数据会按「时间+小区组」进行聚合计算

| 4G计算结果 | `4G指标通报计算结果_YYYYMMDD_HHMMSS.xlsx` | `C:\zhibiao\4G_output\` |
| 5G计算结果 | `5G指标通报计算结果_YYYYMMDD_HHMMSS.xlsx` | `C:\zhibiao\5G_output\` |

**字段说明**：

| 字段名 | 说明 | 计算方式 |

| 小区组 | 小区组名称 | 直接从汇总表取 |

| 高负荷小区数 | PRB利用率≥70%的小区数量 | 从各小区组Sheet统计 |
| 性能劣化小区数 | 低接通+高掉线+低切换（去重） | 从各小区组Sheet统计 |
| 未建立的小区 | 暂无 | 默认0 |

## 六、指标情况计算规则

### 6.1 判定规则表

| 判定类型 | 条件 | 阈值 | 字段 | 数据来源 |
|----------|------|------|------|----------|
| 高负荷小区 | 上行PRB利用率≥70% 或 下行PRB利用率≥70% | ≥70% | 上行PRB利用率-%、下行PRB利用率-% | 各小区组Sheet |
| 低接通小区 | 无线接通率 < 95% | <95% | RRC连接建立成功率_邻区干扰-H(%) | 各小区组Sheet |
| 高掉线小区 | 无线掉线率 > 5% | >5% | 上行掉线率(%) | 各小区组Sheet |
| 低切换小区 | 切换成功率 < 90% | <90% | 切换成功率-D | 各小区组Sheet |
| 性能劣化小区数 | = 低接通 + 高掉线 + 低切换 | 去重 | 按「小区名称」或「基站名称」去重 | 各小区组Sheet |
| 未建立的小区 | 暂无 | 0 | 暂无 | 暂无 |

### 6.2 计算逻辑

1. **高负荷小区**：遍历各小区组Sheet，统计满足「上行PRB利用率≥70%」或「下行PRB利用率≥70%」的小区数量

2. **低接通小区**：统计「RRC连接建立成功率_邻区干扰-H(%) < 95%」的小区数量

3. **高掉线小区**：统计「上行掉线率(%) > 5%」的小区数量

4. **低切换小区**：统计「切换成功率-D < 90%」的小区数量

5. **性能劣化小区数**：将低接通、高掉线、低切换三类小区合并去重（按「小区名称」或「基站名称」去重）

### 6.3 重要提示

⚠️ **指标数据来源于各小区组Sheet，不是汇总表！**

- 汇总表只有9列，不包含PRB利用率等详细指标
- 各项指标需要从各小区组Sheet中按时间点匹配后计算

## 八、常见问题

### Q1：执行时报错"找不到文件"

**原因**：源数据目录或文件不存在

**解决方法**：
1. 确认 `C:\zhibiao\4G_source` 和 `C:\zhibiao\5G_source` 目录存在
2. 确认目录下有 `.xlsx` 文件（不含 `~$` 前缀）

### Q2：指标情况全部为0

**原因**：源数据中各小区组Sheet没有对应的时间点数据

**解决方法**：
1. 检查各小区组Sheet是否包含08:00等时间点的数据
2. 确认列名是否包含「上行PRB利用率」「下行PRB利用率」等

### Q3：计算结果与预期不符

**原因**：可能是版本问题

**解决方法**：
1. 从GitHub获取最新版本：`git pull`
2. 或回退到正确版本：`git log` 查看历史，`git reset --hard commit_id`

### Q4：如何回退到之前的版本？

```bash
# 查看版本历史
git log

# 回退到指定版本（commit_id填入版本号）
git reset --hard commit_id

# 强制推送到远程（谨慎使用）
git push --force
```
